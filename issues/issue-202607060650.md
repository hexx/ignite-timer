# 課題解決プロンプト: oxlint 設定の見直し - ルールの再有効化

## 1. 役割定義 (Persona)
あなたはTypeScriptの静的解析に精通したソフトウェアエンジニアです。プロジェクトの品質を高めるために、リンタールールを適切に構成してください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `.oxlintrc.json` において、`no-explicit-any`, `no-console`, `curly`, `explicit-function-return-type`, `no-magic-numbers` など20近いルールがすべて `"off"` に設定されており、リンターが実質的に無効化されている。
- **再現手順**: `.oxlintrc.json` を開いて確認すれば一目瞭然。
- **関連ファイル/コード**: `/workspace/.oxlintrc.json`

## 3. 解決すべきゴール (Goal & Objective)
- [ ] プロジェクトの性質（Reactフロントエンド + Honoバックエンド + テストコード）に適したルールを選別し、`"warn"` または `"error"` に再有効化する
- [ ] 特に `no-explicit-any` は `"error"` にし、既存コードに `any` が残っている場合は適切に型付けする
- [ ] `curly`（波括弧強制）は `"error"` にし、必要に応じてコードを修正する
- [ ] `no-console` は本番コードでは `"warn"`、テストコードでは `"off"` にする（overrides活用）
- [ ] 再有効化によって発生するlintエラーをすべて修正し、`npx oxlint` が0エラーで通る状態にする

## 4. 期待する成果物 (Output Format)
- 修正後の `.oxlintrc.json`
- ルール再有効化に伴って修正したソースコード
- `npx oxlint` の実行結果（0エラーであることの確認）

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、直ちに現在の `.oxlintrc.json` の解析から始めてください。
