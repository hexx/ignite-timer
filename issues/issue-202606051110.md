# 指示
現在のReact (Vite) + Honoのプロジェクトを、Cloudflare Pages（Pages Functions）のフルスタック構成として完全に最適化し、デプロイ可能な状態にしてください。
以下の4つのステップに沿って、必要なファイルの作成および既存ファイルの修正を行ってください。

## 1. Cloudflare Pages Functionsの構成（エントリーポイントの作成）
- プロジェクトルートに `functions/api/[[route]].ts` を新規作成してください。
- `src/server/index.ts` からHonoアプリ（`app`）をインポートし、`hono/cloudflare-pages` の `handle` 関数を使用して、`export const onRequest = handle(app)` としてエクスポートしてください。
- `src/server/index.ts` で定義されているHonoアプリが、Cloudflare Pagesのルーティング（`/api/*`）と正確に一致するように、必要であれば `new Hono().basePath('/api')` のようにベースパスの設定を調整してください。

## 2. ローカル開発環境の統合（vite.config.ts の修正）
- ローカル開発時（`npm run dev`）に、フロントエンド（React）とバックエンド（Hono）が1つのプロセスでシームレスに動くようにします。
- `vite.config.ts` を修正し、`@hono/vite-dev-server` （およびCloudflare Pages用アダプター）を利用して、Viteの開発サーバー内でHonoアプリを処理できるようにプラグインを追加してください。エントリーポイントは `src/server/index.ts` を指定してください。

## 3. wrangler.toml のクリーンアップと更新
- 現在の `wrangler.toml` に記述されている `[site]` および `bucket = "./dist"` は、レガシーなWorkers Sites用の設定なので削除してください。
- 代わりに、Cloudflare Pagesとしての出力先を明示するため、`pages_build_output_dir = "dist"` を追加してください。（`name` と `compatibility_date` はそのまま維持します）

## 4. package.json の整理（依存関係とスクリプト）
- 上記の実装に必要なパッケージ（`@hono/vite-dev-server` など）を `devDependencies` に追加してください。
- 既存の `"deploy": "wrangler pages deploy dist"` がそのまま機能するように、不要なパッケージやスクリプトがあれば整理してください。
