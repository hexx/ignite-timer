# 指示
現在のコードでは、バックエンド（Hono）に `/api/modes` というエンドポイントが定義されていますが、フロントエンド（App.tsx）ではハードコードされた `modes` 配列を使用しています。
以下の対応を行ってください。

1. App.tsx を修正し、`useEffect` と `fetch` を使って初回マウント時に `/api/modes` からモード情報を取得するようにしてください。
2. 取得中はローディング表示（「読み込み中...」など）を行い、エラー時のハンドリングも追加してください。
3. 必要に応じて、`src/server/index.ts` のレスポンスに `emoji` フィールドを追加するなど、フロントエンドの型に合わせて調整してください。
