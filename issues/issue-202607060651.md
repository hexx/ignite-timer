# 課題解決プロンプト: タイマーコンポーネントの統合 - DRY化

## 1. 役割定義 (Persona)
あなたはReactのコンポーネント設計に長けたフロントエンドエンジニアです。コードの重複を排除し、保守性の高いコンポーネント設計にリファクタリングしてください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `FiveSecondRule.tsx`, `TwoMinuteRule.tsx`, `TenMinuteRule.tsx` の3つのタイマーコンポーネントが、Card → 時間表示 → プログレスバー → ボタン群 というほぼ同一の構造を持ちながら、別々のファイルにコピーされている。異なるのは `duration` の値と、一部の追加機能（5秒はポーズなし、10分は完了後の「続ける」選択肢）のみ。
- **再現手順**: 3つのコンポーネントファイルを見比べれば明らか。
- **関連ファイル/コード**:
  - `/workspace/src/client/components/FiveSecondRule.tsx`
  - `/workspace/src/client/components/TwoMinuteRule.tsx`
  - `/workspace/src/client/components/TenMinuteRule.tsx`

## 3. 解決すべきゴール (Goal & Objective)
- [ ] 3つのコンポーネントを統合し、単一の `TimerCard` コンポーネントを作成する
- [ ] props で duration, ラベル, 説明, ポーズの有無, 完了後アクションなどを設定可能にする
- [ ] 統合後も既存の E2E テスト (`data-testid` の値) がすべてパスすること
- [ ] 統合後も既存の単体テストがすべてパスすること
- [ ] 各モード固有のロジック（5秒の緊急度色分け、10分の「続ける」選択肢）を適切に引き継ぐこと

## 4. 期待する成果物 (Output Format)
- 新規作成された `TimerCard.tsx`（または同等の統合コンポーネント）
- 不要になった古いコンポーネントファイルの削除
- `App.tsx` での使用箇所の更新
- `npm test` および `npm run test:e2e` がパスすることの確認

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、3つのコンポーネントの共通部分と差分の分析から始めてください。
