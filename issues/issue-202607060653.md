# 課題解決プロンプト: TypeScript設定の改善 - ignoreDeprecations / target / paths の見直し

## 1. 役割定義 (Persona)
あなたはTypeScriptの設定とビルド最適化に詳しいソフトウェアエンジニアです。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**:
  1. `tsconfig.json` に `"ignoreDeprecations": "6.0"` が設定されており、TypeScript 6.0 の非推奨API警告がすべて抑制されている。非推奨APIを使っている可能性がある。
  2. `"target": "ES2020"` は React 19 + Vite 8 のモダンスタックに対して古すぎる（ES2022以上が適切）。
  3. `"paths": { "@/*": ["./src/client/*"] }` により、`src/server/` 以下のコードが `@/` エイリアスを使えない非対称な構成になっている。
- **関連ファイル/コード**:
  - `/workspace/tsconfig.json`
  - `/workspace/tsconfig.node.json`

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `ignoreDeprecations` を削除し、発生する非推奨警告を確認・修正する（警告がなければ削除のみでOK）
- [ ] `target` を `ES2022` 以上に更新し、ビルドとテストが通ることを確認
- [ ] paths エイリアスの適用範囲を確認し、必要に応じて server コードも含めるか、server用に別途 tsconfig を用意する
- [ ] `npm run build` および `npm test` がパスすること

## 4. 期待する成果物 (Output Format)
- 修正後の `tsconfig.json`（および関連ファイル）
- 発生した非推奨警告とその修正内容のサマリー
- ビルドとテストの成功確認

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、現在の tsconfig の解析から始めてください。
