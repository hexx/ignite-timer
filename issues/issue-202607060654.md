# 課題解決プロンプト: テストの強化 - E2Eのpage.clock対応と単体テストのエッジケース追加

## 1. 役割定義 (Persona)
あなたはテスト自動化と品質保証に精通したソフトウェアエンジニアです。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**:
  1. E2Eテスト（Playwright）がタイマーの完了を**実際の時間経過**で待っている（5秒タイマーの完了に最大8秒待つなど）。テストが遅く、10分タイマーのテストは事実上書けない。
  2. 単体テスト（Vitest）で以下のエッジケースがカバーされていない：
     - `start()` の2重呼び出し
     - `pause()` を idle 状態で呼んだ場合
     - `resume()` を paused 以外の状態で呼んだ場合
     - `reset()` を各状態で呼んだ場合の一貫性
     - 完了後にもう一度 `start()` した場合
- **関連ファイル/コード**:
  - `/workspace/tests/e2e/timer.spec.ts`
  - `/workspace/tests/unit/useTimer.test.ts`
  - `/workspace/playwright.config.ts`

## 3. 解決すべきゴール (Goal & Objective)
- [ ] Playwrightの `page.clock` API を使用してE2Eテスト内で時間を早送りし、実時間を待たずにタイマー完了をテストできるようにする
- [ ] 10分間ルールのE2Eテスト（完了後の「続ける」「やめる」選択肢のテスト）を追加する
- [ ] 単体テストに上記のエッジケースを追加する
- [ ] 既存のテストがすべてパスすること

## 4. 期待する成果物 (Output Format)
- 修正後の `tests/e2e/timer.spec.ts`
- 修正後の `tests/unit/useTimer.test.ts`
- `npm test` および `npm run test:e2e` の実行結果

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、現在のテストコードの解析から始めてください。
