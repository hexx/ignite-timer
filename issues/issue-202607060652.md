# 課題解決プロンプト: useTimer フックのタイマー精度改善 - Date.now() 方式への移行

## 1. 役割定義 (Persona)
あなたはブラウザのタイミングAPIとReact hooksに詳しいフロントエンドエンジニアです。タイマーの精度問題を解決してください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `useTimer` フックが `setInterval(cb, 1000)` でカウントダウンを実装している。`setInterval` はイベントループの混雑時に遅延し、正確に1秒ごとにコールバックが呼ばれる保証がない。特に10分（600秒）のタイマーでは、実時間とのズレが顕著になる。
- **再現手順**: 10分タイマーを開始し、実際の経過時間と表示のズレを確認する。
- **関連ファイル/コード**: `/workspace/src/client/hooks/useTimer.ts`

## 3. 解決すべきゴール (Goal & Objective)
- [ ] タイマー開始時に基準時刻（`Date.now() + duration * 1000`）を記録し、各tickで残り時間を基準時刻から逆算する方式に変更する
- [ ] tickの間隔は `setInterval` のままでもよいが、表示値は基準時刻から計算する（例: 100ms間隔で表示更新）
- [ ] タブが非表示の間も時間が正しく経過するようにする（`visibilitychange` イベント対応）
- [ ] 既存の単体テストがすべてパスすること（fake timersとの互換性を確保）
- [ ] 一時停止・再開のロジックが正しく動作すること

## 4. 期待する成果物 (Output Format)
- 修正後の `useTimer.ts`
- 必要に応じて追加・修正されたテストケース
- `npm test` がパスすることの確認
- 修正方針の簡潔な解説

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、現在の `useTimer.ts` の解析から始めてください。
