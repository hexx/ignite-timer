# 指示
Cloudflare Pagesにデプロイしたところ、APIが `/api/api/modes` という二重のパスで公開されてしまっていることが判明しました。Cloudflare Pagesのディレクトリベースのルーティングと、HonoのbasePathが重複しているためです。
以下の修正を行って、`/api/modes` で正しくアクセスできるようにしてください。

1. 【Hono側の修正】
`src/server/index.ts` を開き、`new Hono().basePath('/api')` となっている箇所を、単なる `new Hono()` に変更し、basePathの設定を削除してください。

2. 【Functionsの配置確認】
エントリーポイントは `functions/api/[[route]].ts` に配置するのがCloudflare Pagesのベストプラクティスです。もし現在の配置が `functions/[[route]].ts` になっている場合は、`functions/api/[[route]].ts` に移動し、内部のインポートパス（`../../src/server/index` など）を正しく合わせてください。

3. 【ローカル開発環境の修正】
`vite.config.ts` でHonoを動かす設定について、必要であれば `base: '/api'` のようなViteのプロキシ・Devサーバー側の設定とズレがないか確認し、`npm run dev` でも本番でも同じく `/api/modes` でデータが取得できるように調整してください。
