# 課題解決プロンプト: GitHub Actions の CI ワークフロー設定

## 1. 役割定義 (Persona)
あなたは CI/CD と GitHub Actions に精通したシニアソフトウェアエンジニアです。プルリクエストの品質を自動で担保する、堅牢で実用的な CI パイプラインを構築してください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: このリポジトリ（`ignite-timer`）には CI が一切設定されておらず、push/PR 時に lint・型チェック・テストが自動実行されない。そのため、壊れたコードや型エラーが main ブランチに混入するリスクがある。
- **再現手順**: `.github/workflows` ディレクトリを確認すると存在しない。
- **関連ファイル/コード**:
  - `/workspace/package.json`（scripts: `dev`, `build`(`tsc && vite build`), `test`(`vitest run`), `test:e2e`(`playwright test`), `preview`(`wrangler dev`), `deploy`(`wrangler deploy`)）
  - `/workspace/.oxlintrc.json`（oxlint 設定、`npx oxlint` で実行）
  - `/workspace/playwright.config.ts`（e2e は `npm run dev` で `http://localhost:5173` を起動し、chromium でテスト）
  - `/workspace/tsconfig.json`, `/workspace/vite.config.ts`, `/workspace/wrangler.toml`（Cloudflare Workers デプロイ設定）

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `.github/workflows/ci.yml` を新規作成する
- [ ] トリガーは `push`（main ブランチ）と `pull_request`（main 向け）で発火するようにする
- [ ] ジョブ構成は以下の通り（実行環境は `ubuntu-latest`、Node は LTS の安定版 = `20.x` または `22.x` を `actions/setup-node` で指定、`cache: npm` を有効化）:
  - **lint ジョブ**: `npm ci` のち `npx oxlint` を実行し、0エラーであることを確認
  - **typecheck ジョブ**: `npm ci` のち `npx tsc --noEmit` で型エラーがないことを確認
  - **test ジョブ**: `npm ci` のち `npm test`（vitest run）を実行し、単体テストが全て通ることを確認
  - **build ジョブ**: `npm ci` のち `npm run build`（`tsc && vite build`）が成功し `dist/` が生成されることを確認
  - **e2e ジョブ**: ❌ **削除（不要と判断）** — ユーザー要望により e2e テストおよび Playwright は本プロジェクトでは不要と判断し、設定・テスト・依存をすべて削除。詳細は末尾の「追記」を参照。
- [ ] `forbidOnly` / `retries` / `workers` は playwright.config.ts 側で既に `process.env.CI` を見て制御されているため、CI 環境変数はそのまま流用できる
- [ ] （任意・推奨）デプロイステップは CI には含めず、別の `deploy.yml` または手動実行とする。どうしても含める場合は `wrangler deploy` 用の `CLOUDFLARE_API_TOKEN` 等のシークレットが必要である旨をコメントで明記する
- [ ] 各ジョブは独立して並列実行され、全体が成功したときのみマージ可能なステータスチェックとして機能するようにする

## 4. 期待する成果物 (Output Format)
- 新規作成した `.github/workflows/ci.yml`（または分割した複数ワークフロー）
- ワークフローの意図・各ジョブの役割を簡潔に説明した補足
- （可能であれば）ローカルで `act` 等を使って構文・動作を確認した結果、または GitHub 上で実際に workflows が緑になることの確認

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、直ちに `/workspace/package.json` と既存設定ファイルの解析から始めてください。

## 6. 追記（2026-07-09 実行時の変更）
- **e2e / Playwright は不要のため削除済み**。ユーザー要望により、以下を削除した:
  - `.github/workflows/ci.yml` から `e2e` ジョブを削除（残りは `lint` / `typecheck` / `test` / `build` の 4 ジョブ）
  - `playwright.config.ts` を削除
  - `tests/e2e/` ディレクトリ（timer.spec.ts）を削除
  - `package.json` から `"test:e2e": "playwright test"` スクリプトと `@playwright/test` を削除
  - `npm install` を実行し `node_modules` および `package-lock.json` から `@playwright/test` を除去（`@vitest/browser-playwright` は vitest のブラウザ機能依存として残存し、e2e 用ではない）
- **検証結果**: `npx oxlint`（0 エラー）、`npx tsc --noEmit`（No errors）、`npm test`（vitest 10 passed）、`npm run build`（dist 生成）を確認。CI は緑になる構成。
