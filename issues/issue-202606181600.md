# 指示
現在のプロジェクト（Hono + React + Vite）を、Cloudflare Pages（Pages Functions）の構成から、単一の「Cloudflare Workers」として動作し、かつReactの静的ファイルも配信するフルスタック構成に移行してください。

以下のステップに従ってファイルを作成・修正・削除してください。

## Step 1: バックエンドのエントリーポイントの変更と不要ファイルの削除
1. Cloudflare Pages専用のエントリーポイントである `functions/[[route]].ts` （および `functions` ディレクトリ全体）を削除してください。
2. Honoのメインファイルである `src/server/index.ts` を開き、Cloudflare Pages特有の処理（`hono/cloudflare-pages` など）があれば削除し、純粋なCloudflare Workersのエントリーポイントとして機能するように `export default app` となっていることを確認・調整してください。

## Step 2: 静的ファイル（Reactフロントエンド）の配信設定
Cloudflare Workers上でAPI（`/api/*`）以外のリクエストが来た際に、Reactの静的ビルドファイル（`dist` ディレクトリ）を返すようにする必要があります。
Wranglerの新しい静的アセット機能（Workers Assets）を利用することを前提とします。

## Step 3: `wrangler.toml` のWorkers向け最適化
`wrangler.toml` を以下の内容に更新してください。
- `pages_build_output_dir = "dist"` を削除してください。
- Workersのエントリーポイントとして `main = "src/server/index.ts"` を追加してください。
- Reactのビルド出力先を静的アセットとして指定するため、以下の設定を追加してください。
  ```toml
  [assets]
  directory = "dist"
  binding = "ASSETS"
  ```

※ `name` と `compatibility_date` はそのまま維持してください。

## Step 4: `package.json` のスクリプトの更新

* `package.json` 内の `"deploy": "wrangler pages deploy dist"` というスクリプトを、Workers用の `"deploy": "wrangler deploy"` に変更してください。
* その他、Pages特有の依存パッケージが残っていれば整理してください（必須ではありません）。

## Step 5: ローカル開発環境の確認

* `vite.config.ts` で設定されている `@hono/vite-dev-server` の設定が、Workers構成移行後も `npm run dev` で正しく動作するか確認し、必要に応じてアダプター設定をCloudflare Workers向けに調整してください。
